# Historial-de-usuarios-y-el-flujo-
Caso Práctico: "FastPass Uni: Pide, Paga y Recoge tu Almuerzo"

solucion:

Story #1: Ver el menú disponible
Como estudiante que quiere almorzar rápido,
quiero ver el menú disponible de todos los puntos de comida en la app,
para decidir qué pedir antes de salir del salón.

User Story #2: Pedir comida desde el celular
Como estudiante que tiene poco tiempo de almuerzo,
quiero pedir mi comida desde la aplicación,
para evitar hacer filas largas en la cafetería.

User Story #3: Pagar desde la aplicación
Como estudiante que compra en la cafetería,
quiero pagar mi pedido desde la app,
para recoger mi comida sin tener que esperar en caja.

User Story #4: Recibir notificación del pedido
Como estudiante que hizo un pedido,
quiero recibir una notificación cuando mi comida esté lista,
para pasar a recogerla en el momento adecuado.

User Story #5: Ver disponibilidad del menú del día
Como estudiante que quiere el menú del día,
quiero ver si aún está disponible en la app,
para no hacer un pedido de algo que ya se agotó.

User Story #6: Recoger el pedido rápidamente
Como estudiante con poco tiempo de almuerzo,
quiero recoger mi comida mostrando mi pedido en la app,
para volver rápido a clase.

2. como segundo punto

Historia de Usuario 1: Ver el menú disponible
Requerimientos funcionales:
1.	El sistema debe mostrar el menú de todos los puntos de venta de la universidad.
2.	El sistema debe mostrar nombre del plato, precio y disponibilidad.
3.	El sistema debe permitir actualizar el menú en tiempo real cuando un producto se agote.

Historia de Usuario 2: Pedir comida desde el celular
Requerimientos funcionales:
1.	El sistema debe permitir al estudiante seleccionar productos del menú.
2.	El sistema debe permitir agregar varios productos al pedido.
3.	El sistema debe permitir confirmar el pedido antes de enviarlo.

Historia de Usuario 3: Pagar desde la aplicación
Requerimientos funcionales:
1.	El sistema debe permitir realizar el pago del pedido desde la app.
2.	El sistema debe confirmar si el pago fue exitoso o rechazado.
3.	El sistema debe generar un comprobante de pago digital.

Historia de Usuario 4: Recibir notificación del pedido
Requerimientos funcionales:
1.	El sistema debe notificar al estudiante cuando el pedido esté listo.
2.	El sistema debe mostrar el estado del pedido (en preparación, listo, entregado).
3.	El sistema debe enviar notificaciones al celular del usuario.

Historia de Usuario 5: Ver disponibilidad del menú del día
Requerimientos funcionales:
1.	El sistema debe mostrar el menú del día disponible.
2.	El sistema debe indicar cuando un producto esté agotado.
3.	El sistema debe actualizar automáticamente la disponibilidad del menú.

Historia de Usuario 6: Recoger el pedido rápidamente
Requerimientos funcionales:
1.	El sistema debe generar un código o comprobante del pedido.
2.	El sistema debe permitir al personal verificar el pedido al momento de la entrega.
3.	El sistema debe marcar el pedido como entregado en el sistema.
